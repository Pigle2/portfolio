## 작업 룰

- **콘텐츠는 docs/에.** git 관리, 다른 머신에서도 보임. 메모리는 로컬 보조 포인터만.
- **떠넘기지 말 것.** 정해진 정보로 추론 가능하면 작업자가 제안, 사용자는 검토.
- **객관식 금지.** 옵션 던지지 말고 주관식 자유 답.
- **이미 말한 것 다시 묻지 말 것.** 누적 답변 먼저 확인.
- **단어 ≠ 의미.** 단어 자체로 평가 X. 그 단어가 전달할 의미가 사실에 부합하는지로 평가.
- **진정성 우선.** 임팩트/호기심/도발/후킹 같은 잡기술 평가 금지. 사실 + 증명 + 무꾸밈만.
- **비격식 금지.** "굴리다 / 좀 / 그냥 / 막 / 이딴" 같은 구어 본문 금지.
- **경력 ≠ 경험.** 경력(회사/직책/기간)은 이력서로 위임. 경험(사례/결과/만든 것)은 포트폴리오 본문에 살아있어야 함. 둘을 같은 카테고리로 묶지 말 것.
