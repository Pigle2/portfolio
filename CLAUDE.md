면접관에게 이 URL을 공유하여 경력/프로젝트/AI 활용 사례를 보여줌.

## 기술 스택
- HTML / CSS / JS (순수, 프레임워크 없음)
- Google Fonts (Noto Sans KR)
- GitHub Pages 배포

## 사이트 구조
1. **Hero** — 메인 타이틀, 포지셔닝
2. **About** — 자기소개, 경력 요약 통계
3. **Skills** — 기획 역량 / 개발 역량 / AI & 도구
4. **Career Timeline** — 개발자 → 기획자 → AI 기획자 전환 타임라인
5. **Projects** — 첫차(100만DL), 증권 HTS/MTS, 붕붕마켓
6. **AI Showcase** — AI 활용 사례 (이 사이트, Agent Teams, N8N)
7. **Contact** — 연락처, Figma 링크

## 메인 문구
- 1줄: "개발을 이해하고 AI로 일하는 기획자"
- 2줄: "빠른 생산성, 프로토타입까지 직접 구현하는 인재"
- 설명: "기술 실현 가능성을 판단하고, AI 에이전트로 업무를 자동화합니다"

## 포지셔닝 제약조건 (중요)
- 임상준은 현재 코드를 읽지 않고, API를 설계하지 않음
- "코드를 읽고", "API를 설계하는", "코드 레벨 분석", "코드 리뷰" 등의 표현 사용 금지
- 100만 DL은 백엔드 개발자 시절의 성과이지, PM으로서의 성과가 아님
- 100만 DL, 1000억 거래 등 과거 성과를 대표 문구/타이틀/태그라인으로 사용 금지. 본문 내 자연스러운 언급은 허용.
- 대표 문구는 "뭘 할 수 있는지"(역량)를 어필해야 함. "뭘 했느냐"(과거 성과) 중심 금지.
- 올바른 표현: "개발 경험을 바탕으로", "7년간의 서버 개발 경험으로", "개발을 이해하는 기획자"
- 현재 역할을 과장하지 말 것. 과거 경험은 정확한 역할로 귀속할 것.
