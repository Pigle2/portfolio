## 4. 포지셔닝 제약조건 (중요)
콘텐츠 작성 시 반드시 준수해야 하는 규칙. 위반 시 기획 의도와 어긋남.

### 4.1 금지 표현
- "코드를 읽고", "API를 설계하는", "코드 레벨 분석", "코드 리뷰"
- "Technical PM"
- "프롬프트 엔지니어링" (할 줄 모름 — AI 관련 기술에서 빠져야 함)
- "코드로 기획"
- 100만 DL / 1000억 거래 등 과거 성과를 대표 문구 · 타이틀 · 태그라인으로 사용 금지
  (본문 내 자연스러운 언급은 허용)

### 4.2 올바른 표현
- "개발 경험을 바탕으로"
- "7년간의 서버 개발 경험으로"
- "개발을 이해하는 기획자"

### 4.3 원칙
- 임상준은 현재 코드를 읽지 않고, API를 설계하지 않음. 코드를 보지 않고, 보고 싶지도 않음.
- 100만 DL은 백엔드 개발자 시절의 성과이지, PM으로서의 성과가 아님
- 대표 문구는 "뭘 할 수 있는지"(역량) 중심. "뭘 했느냐"(과거 성과) 중심 금지
- 현재 역할은 "기획자"이지 "Technical PM"이 아님
- 현재 역할을 과장하지 말 것. 과거 경험은 정확한 역할로 귀속할 것
