## 1. 목적
임상준의 웹 버전 이력서. 구직 실전용.
포트폴리오 v3("AI로 프로젝트를 진행하는 사람")와 정체성을 정렬한다.
이력서의 1차 임무는 "만나보고 싶다"를 만드는 것.

## 2. 타깃
- 이력서를 직접 검토하는 실무자 / 면접관
- 프린트 가능한 A4 레이아웃 제공 (인쇄용 스타일 포함)

## 3. 접근법
**Approach A: 미니멀 리프레이밍.** 현재 resume.html 구조를 유지하고 텍스트만 수정. 프로필 사진 추가.

와이어프레임 비교(A/B/C) 후 사용자 최종 선택.

## 4. 구성

```
1. 프로필 사진 (111.jpg) + 이름 + 한 줄 정체성 선언
2. 인적사항 테이블
3. Summary
4. 핵심 역량
5. 경력사항 — 기획
6. 백엔드 개발 경력 (7년)
7. 주요 프로젝트
8. 기술 스택
9. 학력
10. 취업우대사항
11. 자기소개서
```

## 5. 변경 항목 (v1 → v2)

### 5.1 추가: 프로필 사진
- `111.jpg`를 최상단(이름 옆)에 배치
- 인쇄 시에도 표시

### 5.2 변경: 정체성 선언 (blockquote)
- **Before:** "개발 경험을 바탕으로 기술과 비즈니스를 연결하는 서비스 기획자"
- **After:** v3 정체성에 맞는 문구로 교체 (구체 문구는 아래 6번에서 확정)

### 5.3 변경: Summary
- AI 활용을 "도구 나열"에서 "프로젝트 진행 방법론"으로 격상
- 포트폴리오 사이트 자체가 AI로 만든 결과물임을 언급

### 5.4 변경: 핵심 역량
- "AI 도구 활용" → "AI 프로젝트 실행"으로 교체
- 내용: Claude Code 기반 기획, 개발, 교차 검증, QA 전 과정 1인 수행
- N8N 제거 (포트폴리오-기획서 4.7 제외 항목)

### 5.5 변경: 경력사항 bullet
- 팩트(회사/기간/역할) 유지
- 설명 문구를 임팩트/성과 언어로 리프레이밍

### 5.6 변경: 기술 스택
- AI 도구에서 N8N 제거
- 협업 도구에서 Antigravity 제거

### 5.7 유지: 자기소개서
- "사람 냄새나는" 톤 유지. 큰 변경 없음.
- v3 정체성과 직접 충돌하는 문장만 교체
- 톤/인격 표현은 변경 금지
- 비격식 금지 규칙은 이 섹션에 의도적 예외

## 7. 작성 원칙
**포트폴리오 기획서 v3.0.7의 포지셔닝 제약조건과 동일한 규칙을 적용한다.**
→ `docs/포트폴리오-기획서.md` **5. 포지셔닝 제약조건** 섹션 참조

요약:
- "Technical PM", "프롬프트 엔지니어링", "코드로 기획" 등 금지 표현 사용 안 함
- 100만 DL / 1000억 거래 = 본문 내 언급만 허용, 대표 문구 금지
- 현대차 그룹사 시작일 2025.10
- 루덴스플레이 / 플라이북 별도 표기
- N8N, Antigravity = 제외

**경력 ≠ 경험 분리 원칙:**
- 경력(회사/직책/기간) = 이력서
- 경험(사례/결과/만든 것) = 포트폴리오
- 겹치지 않게
