Case Study: 코드 읽는 기획자

증권 HTS — 배당률 로직 오류 발견

출시 전 장애 예방
코드 리뷰 기획자가 직접
QA 단축 커뮤니케이션 공수 절감

증권 HTS/MTS/WTS 모바일 생체 인증 로그인 기획 중, 배당률 계산 로직에 오류가 있는 것을 코드 리뷰로 직접 발견했습니다. 기획자 → 개발자 → QA의 일반적 프로세스가 아닌, 기획자가 직접 코드를 분석하여 문제를 짚어내고 수정 방향을 제시한 사례입니다.

서비스 기획 코드 레벨 분석 정책 설계 QA
Figma, Notion, Jira, 증권 거래 시스템
왜 이 사례가 중요한가: 기획자가 코드를 읽으면 — 개발팀과 동일한 언어로 소통하고, 로직 오류를 QA 단계 이전에 잡아내며, 커뮤니케이션 왕복을 줄입니다. 이 경험은 "기획자가 왜 코드를 읽어야 하는가"에 대한 실전 증명입니다.

Paperclip Claude Code Multi-Agent

AI 멀티 에이전트 회사 운영 시스템

Paperclip 프레임워크 위에 CEO, CTO, PM 등 역할별 AI 에이전트로 구성된 가상 회사를 설계하고 운영합니다. 에이전트들이 이슈를 주고받으며 자율적으로 업무를 수행하고, 사람이 의사결정만 하는 구조입니다.

내가 한 것

  • 에이전트 역할 체계 설계 (CEO → CTO → IC 위임 구조)
  • 하트비트 기반 자율 실행 시스템 구축
  • 이슈/태스크 관리, 예산 거버넌스 정책 수립
  • 에이전트간 커뮤니케이션 프로토콜 설계
이것이 증명하는 것: AI를 개별 도구가 아닌 조직 시스템으로 설계하고, 거버넌스와 자율성을 균형 있게 운영할 수 있는 역량. AI PM의 핵심 역할인 "AI 시스템 오케스트레이션"의 실전 경험.
